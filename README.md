# Watch Display Defect Analysis

Watch Display의 R/G/B/W 점등 이미지를 불러와 학습된 불량 데이터와 매칭하고, 결과를 대시보드로 시각화하는
**오프라인 단일 HTML 앱**이다.

설치, 서버, 인터넷 연결이 필요 없다. 이미지는 브라우저 안에서만 처리되며 네트워크로 전송되지 않는다.

## 현재 상태

**전체 5단계 구현 완료 (MVP).** 모든 검증은 합성 이미지 기준이다.

1단계 — 전처리:

- 원형 Active 영역 자동 검출 (Otsu → 최대 연결 성분 → 구멍 채우기 → 최소제곱 원 피팅)
- FPCB 방향 자동 추정 및 신뢰도 평가
- FPCB 6시 정렬 회전 보정 (사용자 확정 필수)
- 512×512 정규화 프레임 리샘플링 및 Active 마스크 생성
- 파일명 기반 패널 자동 그룹핑, 결손·중복·파싱 실패 처리
- 검출 결과 오버레이 UI, 수동 원 지정, 회전각 조정
- 합성 패널 생성기 — 실제 이미지 없이 파이프라인 검증용

2단계 — Rule 검출:

- 전역 강건 2차 곡면 배경 모델 (Otsu 사전 분할 + Tukey IRLS)
- 암점 면적 등급 판정 (小/中/大), 명점, 명선·암선 가로/세로
- 미점등 (전체/부분), 구동불량 의심 플래그
- 패턴 교차 확인 — 2개 이상 패턴에서 보여야 인정, 아니면 검수 대상
- 복수불량 판정 및 severity 기반 Primary Defect 선정
- 패널별 판정 근거(`decision_reason`)와 신뢰도 표시

임계값 튜닝:

- 13개 임계값 슬라이더. 값을 바꾸면 검출 박스가 즉시 갱신되는 실시간 미리보기
- 잘못된 조합(암점 中 상한 ≤ 小 상한 등)은 자동 보정하고 사유를 알림
- JSON 내보내기 / 가져오기, 기본값 복원, localStorage 자동 저장
- 임계값 변경 시 기존 분석 결과를 stale로 표시

3단계 — 라벨링 및 영속화:

- IndexedDB 영속화. 패널·이미지·라벨이 브라우저를 닫아도 유지됨
- 정규화 프레임 위에 점/선/박스로 불량 위치 라벨링
- 업로드 시 용도(학습용/분석용)·Model·Process·Equipment·작업자 메타데이터 입력
- 패널 승인 워크플로 (승인 시 라벨이 approved로 전환)
- 라벨 JSON 내보내기/가져오기 (이미지 픽셀 제외, 라벨·기하만)
- 이미지 Blob 저장, 재분석 없이 재실행 가능하도록 검출 결과 캐시

4단계 — kNN 유사도 검색 및 융합 판정:

- 패턴 이미지당 91차원 디스크립터(방사·각도 히스토그램 + 전역 통계), 4패턴 concat → 364차원 L2 정규화
- 정규화 프레임이 6시 정렬돼 있어 벡터가 회전 정렬된 위치 정보를 담음
- 승인된 학습 패널만 IndexedDB embedding store에 저장(feature_version 태깅)
- brute-force 코사인 유사도 검색, 유사도 가중 다수결
- Rule + kNN 융합: Rule이 불량을 판정하면 우선, 양품이면 kNN이 불량 승격 가능(구동불량 경로). 불일치는 검수 대상
- 학습 패널 수가 임계 미만이면 kNN 비활성, Rule 단독 판정

5단계 — 대시보드:

- 개요 KPI (전체/양품/불량/불량률/복수불량/검수필요/Top불량)
- 불량 비율 (Panel 기준 / Label 기준 토글, 분모 명시)
- 위치 상관: 면적 정규화 원형 Heatmap, DBSCAN Hotspot(패널 수 기준 정렬), 원형 통계 방향 집중도(Rayleigh 검정), 불량명×Region 카이제곱 교차표
- 패턴 분석 R/G/B/W 히트맵, Lot/설비/공정별 불량률
- Lot·설비·공정 필터, 검수 완료만 토글
- ECharts 대신 순수 Canvas — 오프라인 자기완결성 유지, 표본 부족 시 명시

```bash

npm run dev      # 개발 서버
npm test         # 198개 테스트
```

서버·인터넷·설치가 필요 없다.

## 개요

앱은 두 개의 독립된 기능 영역으로 구성된다.

- **학습**: 불량 Master 관리, 학습 이미지 등록, 위치 라벨링, 승인
- **분석**: 이미지 일괄 등록, 자동 불량 매칭, 검수, 대시보드

매칭 엔진은 결정론적 Rule 검출과 kNN 유사도 검색을 결합한다. 두 판정이 불일치하면 검수 큐로 올라가고,
사람의 수정이 곧 다음 학습 데이터가 된다. 딥러닝 프레임워크와 GPU를 쓰지 않는다.

# 실제 이미지 튜닝 절차

전체 흐름은 **전처리 확인 → 검출 감도·경계 조정 → 임계값 내보내기 → 학습 데이터 축적 → 분석·대시보드** 순서다.
반드시 전처리부터 확인한다. 앞 단계가 틀리면 뒤 단계 조정은 전부 헛수고가 된다.

## 0. 시작하기 전에

1. 회사 메일로 받은 `index.html` 을 PC에 저장하고 **더블클릭**한다. Chrome 또는 Edge 권장.
   설치·서버·인터넷이 필요 없고, 이미지는 브라우저 밖으로 나가지 않는다.
2. 처음 한 번은 브라우저가 "이 사이트가 저장소를 사용하도록 허용" 같은 안내를 띄울 수 있다. 허용한다.
3. **이 앱의 모든 데이터(이미지·라벨·임계값)는 이 브라우저에만 저장된다.** 브라우저 캐시/사이트
   데이터를 지우거나, 시크릿 모드로 열거나, 다른 브라우저로 열면 데이터가 보이지 않는다. 자세한 백업은
   맨 아래 "데이터 안전" 참고.

## 1. 전처리부터 확인한다 (가장 중요)

원 검출과 FPCB 회전 정렬이 틀리면 검출·위치·대시보드가 전부 무의미해진다. 임계값을 만지기 전에 이것부터 본다.

1. 용도를 **분석용**으로 두고, 점등 이미지를 드래그해 넣는다.
   - 파일명이 `{LOT_ID}_{PANEL_CODE}_{PATTERN}.png` 형식(예: `LOT2401_P001_R.png`)이면 R/G/B/W가 한 패널로 자동 묶인다.
   - 형식이 다르면 각 이미지가 개별 패널로 들어온다. 지금은 그대로 둬도 검출·튜닝은 가능하다.
2. 패널의 아무 썸네일이나 클릭해 상세 화면을 연다. 왼쪽은 원본 + 검출된 원(초록 점선)과 FPCB 추정 방향(주황),
   오른쪽은 6시로 정렬된 정규화 프레임이다.
3. 다음을 눈으로 확인한다.
   - **초록 원이 Active 디스플레이 경계에 맞는가?** 어긋나면 상세 화면 아래 회전각 슬라이더 옆의
     원 중심 X/Y·반지름 슬라이더로 직접 맞춘다 (자동 검출 실패 시 자동으로 나타난다).
   - **오른쪽 정규화 프레임에서 FPCB 탭이 하단(6시)에 오는가?** 아니면 **회전각** 슬라이더로 맞춘다.
   - 상태 줄에 `FPCB 자동 추정이 불확실합니다`가 뜨면 자동 추정을 믿지 말고 직접 6시에 맞춘다.
4. 맞으면 **회전각 확정**을 누른다. 같은 패널의 다른 패턴도 같은 값이면 **패널 전체에 적용**을 먼저 누른다.
5. 모든 패널에 대해 반복한다. 요약 줄의 `회전각 확정 N/M`이 전부 확정될 때까지.

> 팁: 원 검출이 자주 실패하면 촬영 배경이 너무 밝거나 디스플레이가 화면을 꽉 채웠을 가능성이 크다.
> 그런 이미지 몇 장을 따로 알려주면 검출 로직을 조정하겠다.

## 2. 검출 감도를 맞춘다

상세 화면 아래 **임계값 조절**을 펼친다. 슬라이더를 움직이면 오른쪽 정규화 프레임의 **검출 박스가 즉시
갱신**되고, 상단에 `dark_area_ratio`와 검출된 성분 수가 실시간으로 표시된다. 이 화면을 보며 맞춘다.

증상을 보고 슬라이더를 고른다.

| 증상 | 조정할 슬라이더 | 방향 |
|---|---|---|
| 멀쩡한 부분이 암점으로 잔뜩 잡힘 | **암부 잔차 임계** | 올린다 |
| 실제 암점이 안 잡힘 | **암부 잔차 임계** | 내린다 |
| 멀쩡한 부분이 명점으로 잔뜩 잡힘 | **명부 잔차 임계** | 올린다 |
| 실제 명점이 안 잡힘 | **명부 잔차 임계** | 내린다 |
| 아주 작은 점 노이즈가 계속 잡힘 | **최소 검출 면적** | 올린다 |
| 미세한 실제 불량이 사라짐 | **최소 검출 면적** | 내린다 |
| 특정 색에서만 보이는 불량이 무시됨 | **패턴 교차 확인 수** | 내린다 (1이면 교차 확인 끔) |
| 촬영 노이즈가 불량으로 승격됨 | **패턴 교차 확인 수** | 올린다 |

기준 잡는 법: **양품 패널 몇 장을 먼저 연다.** 양품인데 검출 박스가 뜨면 그 종류의 잔차 임계를 박스가
사라질 때까지 올린다. 그다음 **불량 패널**을 열어 실제 불량이 잡히는 가장 높은 임계를 찾는다. 두 지점
사이가 안전 구간이다.

## 3. 등급·형상 경계를 맞춘다

감도가 맞으면 판정 경계를 본다. 이 값들은 이미지가 아니라 **판정 기준**이라, 실제 불량 몇 개를 열어
`dark_area_ratio` 실측값과 판정을 비교하며 맞춘다.

- **암점 小/中 상한 (%)**: 상세 화면 상단 `dark_area_ratio`가 실제 암점 면적비다. 현장에서 中으로
  분류하는 암점의 실측 비율을 보고 **암점 小 상한**과 **암점 中 상한**을 그 사이로 옮긴다.
  (中 상한은 항상 小 상한보다 커야 하며, 작게 넣으면 자동 보정된다.)
- **Line 최소 종횡비 / 최소 길이**: 짧은 얼룩이 선으로 오인되면 두 값을 올린다. 실제 가는 선이 점으로
  분류되면 내린다.
- **가로/세로 허용각**: 비스듬한 선까지 가로/세로로 인정하려면 올리고, 엄격히 수평·수직만 인정하려면 내린다.
- **전체 미점등 평균 휘도**: 미점등 패널의 실제 평균 휘도(상태 줄에 표시)보다 약간 높게 잡는다.
- **부분 미점등 면적비**: "이 정도 어두우면 암점 大가 아니라 미점등"의 경계. 기본 0.6(60%).
- **center/mid 상한 반경비**: 위치를 center/mid/edge로 나누는 경계. 현장 관행에 맞춘다.

## 4. 임계값을 저장하고 공유한다

1. **임계값 조절** 하단의 **JSON 내보내기**를 누른다 → `defect-thresholds-날짜.json` 다운로드.
2. 이 파일을 **나(개발자)에게 전달하면**, 실제 이미지를 못 보는 상태에서도 기본값을 실물에 맞게 갱신할 수 있다.
3. 다른 PC나 동료는 같은 파일을 **JSON 가져오기**로 불러 동일한 설정을 재현한다.
4. 잘못된 조합(예: 암점 中 상한 ≤ 小 상한)은 가져올 때 자동 보정되고 무엇을 고쳤는지 알려준다.

임계값은 브라우저(localStorage)에도 자동 저장되어 앱을 다시 열어도 유지된다. 하지만 브라우저 데이터를
지우면 사라지므로, **내보낸 JSON이 유일한 백업이다.**

## 5. 학습 데이터를 만든다 (kNN 정확도의 근거)

Rule 검출만으로는 구동불량처럼 못 잡는 불량이 있다. kNN이 이를 보완하려면 **사람이 라벨링해 승인한
학습 패널**이 필요하다.

1. 용도를 **학습용**으로 바꾸고 대표적인 불량 이미지를 불러온다.
2. 패널 썸네일을 열고, 1~2절대로 원·회전을 확정한다.
3. **불량명**을 고르고 **도구**(점/선/박스)를 선택해 정규화 프레임 위에 불량 위치를 그린다.
   - 점: 클릭. 선·박스: 드래그. Active 영역 밖은 무시된다.
   - 그린 라벨은 아래 목록에 위치(시계 방향·region)와 함께 나타난다. 잘못 그렸으면 **삭제**.
4. 라벨링이 끝나면 패널 헤더의 **승인**을 누른다. 승인 시 그 패널의 특징 벡터가 kNN 검색 대상에 즉시 추가된다.
5. 학습 패널이 **10개 이상**(기본값, kNN 활성 최소 학습 수) 쌓이면 분석 시 kNN이 동작한다. 그 전에는
   Rule 단독으로 판정하고 그 사실을 표시한다.
6. **라벨 내보내기**로 라벨·기하를 JSON으로 저장해 백업하고 나에게 전달한다. (이미지 픽셀은 보안상 제외되며,
   따라서 가져온 라벨 패널은 재분석되지 않는다.)

kNN 관련 임계값(**최소 유사도**, **이웃 수 k**, **kNN 활성 최소 학습 수**)도 같은 슬라이더 패널의
`kNN 매칭` 그룹에서 조정할 수 있다. 학습 데이터가 적을 때 동작을 보려면 **kNN 활성 최소 학습 수**를
잠시 낮춰도 된다.

## 6. 분석하고 대시보드로 본다

1. 분석용 이미지를 모두 불러오고 원·회전을 확정한 뒤 **불량 분석 실행**을 누른다.
2. 각 패널 카드에 AI 판정·신뢰도·판정 근거가, Rule과 kNN이 불일치하면 **검수 필요** 배지가 뜬다.
3. **대시보드**를 누르면 불량 비율, 위치 상관(원형 Heatmap·Hotspot·방향 집중도), 패턴 분석,
   Lot/설비/공정별 불량률을 볼 수 있다. Lot·설비·공정 필터로 좁혀 본다.

## 데이터 안전

- 모든 데이터는 이 브라우저의 IndexedDB/localStorage에만 있다. **서버 백업이 없다.**
- 다음 경우 데이터가 사라진다: 브라우저 사이트 데이터 삭제, 시크릿 모드 종료, 브라우저 재설치, 다른 브라우저 사용.
- **주기적으로 라벨 내보내기와 임계값 내보내기를 해서 JSON을 별도 보관하라.** 이것이 유일한 백업 수단이다.
- 여러 PC에서 작업하려면 각 PC에서 내보낸 JSON을 가져오기로 합친다.

## 나에게 무엇을 보내면 되나

실제 이미지를 못 보는 상태에서 로직을 실물에 맞추려면 다음이 도움이 된다.

1. **임계값 JSON** (`defect-thresholds-*.json`) — 현장에서 맞춘 값.
2. **라벨 JSON** (`defect-labels-*.json`) — 라벨·기하 (이미지 없음, 반출 안전).
3. **어긋난 지점 메모** — 원 검출 실패가 잦은 조건, 특정 불량이 잘 안 잡히거나 과검출되는 상황 등.

이 세 가지면 이미지 자체 없이도 원 검출·임계값·검출 로직을 실물 기준으로 조정할 수 있다.
