# Dashboard Specification

대시보드는 **승인 여부와 무관하게 분석이 끝난 패널 전체**를 기본 모수로 한다.
`검수 완료만 보기` 토글로 `review_status='approved'` 만 필터링할 수 있다.

## 0. Global Filters

모든 섹션에 공통 적용된다. 변경 시 전체 재계산한다.

| 필터 | 타입 | 기본값 |
|---|---|---|
| Upload Batch | multi-select | 최근 배치 1개 |
| 기간 | date range | 전체 |
| Lot ID | multi-select | 전체 |
| Model | multi-select | 전체 |
| Process | multi-select | 전체 |
| Equipment ID | multi-select | 전체 |
| 검수 완료만 | toggle | off |

현재 필터로 선택된 패널 수를 항상 헤더에 노출한다. `N = 128 panels`

## 1. Overview

KPI 타일 7개.

| 타일 | 정의 |
|---|---|
| 전체 Panel 수 | `count(panel)` |
| 양품 수 | `count(final_judgement_id = 'D000')` |
| 불량 Panel 수 | `전체 - 양품` |
| 불량률 | `불량 Panel 수 / 전체 × 100` |
| 복수불량 수 | `count(final_judgement_id = 'D011')` |
| Top 불량 | 아래 2절 **Label 기준** 최다 `defect_name` |
| 주요 집중 위치 | 4절 hotspot 중 패널 수 1위 클러스터의 시계 방향 표기 (예: `7시 방향 edge`) |

검수가 필요한 패널이 있으면 헤더에 배너를 띄운다.
`검수 필요 12건 — Rule과 kNN 판정이 불일치합니다` → 클릭 시 Review 화면으로 이동.

## 2. Defect Ratio

**두 가지 모수를 반드시 구분해 제공한다.** 이 구분이 없으면 수치가 오독된다.

### Panel 기준

분모 = 전체 패널 수. 분자 = 해당 불량이 **최종 판정**인 패널 수.
복수불량 패널은 `D011` 한 곳에만 계수된다. 따라서 **모든 비율의 합이 100%**다.

```ts
countBy(panelResults, r => r.final_judgement_id)
```

### Label 기준

분모 = 검출된 전체 라벨 수. 분자 = 해당 `defect_id` 라벨 수.
복수불량 패널의 개별 불량이 각각 계수된다. `D000`, `D011`은 등장하지 않는다.
**합이 100%지만 패널 수와는 일치하지 않는다.**

```ts
countBy(annotations, a => a.defect_id)
```

### 표시

- 가로 막대 차트, 수량 내림차순. 막대 끝에 `수량 (비율%)` 라벨.
- `Panel 기준 / Label 기준` 세그먼트 토글. 전환 시 부제로 분모를 명시한다: `분모: 128 panels` / `분모: 173 labels`
- 도넛 차트는 쓰지 않는다. 불량 클래스가 10개를 넘어 각도 비교가 불가능하다.

## 3. Panel Matching Result

가상 스크롤 테이블. 요청하신 **'이미지-불량명' 매칭 내용**이 여기에 해당한다.

| 컬럼 | 내용 |
|---|---|
| Panel ID | `lot_id / panel_code` |
| R / G / B / W | 썸네일 4개. 검출된 불량 위치에 마커 오버레이 |
| 검출 불량 | `detected_defect_ids` 를 칩으로 나열 |
| 최종 판정 | `final_judgement_name`. 복수불량은 강조 |
| 신뢰도 | 0~1 막대. `< 0.6` 이면 주의색 |
| 위치 | Primary defect의 `7시 · edge` 표기 |
| 검수 상태 | pending / approved / rejected 배지 |

동작:

- 행 클릭 → 상세 패널. 정규화 프레임 원본 + 마스크 오버레이 + `decision_reason` + **kNN 이웃 썸네일 5개**를 나란히 보여준다.
- `needs_review = 1` 인 행은 상단에 고정한다.
- 정렬: 신뢰도 오름차순이 기본. 사람이 봐야 할 것부터 올라온다.
- CSV / XLSX 내보내기.

## 4. Location Correlation

요청하신 **'이미지들 간 불량위치 상관성'** 이다. 네 가지 뷰로 구성한다.

모든 좌표는 FPCB 6시 정렬된 정규화 프레임 기준이므로 패널 간 직접 비교가 가능하다.

### 4.1 원형 Heatmap (Polar Density)

- 극좌표 비닝: 각도 24 섹터(15°) × 반경 6 링 = 144 셀
- 셀 값 = 해당 셀에 대표점이 들어간 `annotation` 수
- **셀 면적으로 정규화한다.** 바깥 링일수록 면적이 크므로, 정규화하지 않으면 외곽이 무조건 진하게 보인다.
  ```text
  density = count / cell_area_ratio
  cell_area_ratio = (r_out² - r_in²) / (24 섹터)
  ```
- 6시 방향(0°)에 FPCB 마커를 고정 표시하고, 3·6·9·12시 기준선을 그린다
- 불량명 필터로 특정 클래스만 볼 수 있다

### 4.2 Hotspot Cluster

**"여러 패널에서 같은 자리에 반복되는 불량"** 을 찾는다. 공정·설비 원인 규명의 핵심 뷰다.

```text
입력: 필터된 전체 annotation 의 (x, y) — 정규화 프레임
좌표: u = (x - 256) / 250,  v = (y - 256) / 250     # 단위원
알고리즘: DBSCAN(eps = 0.08, min_samples = 3)        # eps = Active 반지름의 8%
```

클러스터마다 다음을 테이블로 제시한다.

| 컬럼 | 내용 |
|---|---|
| Hotspot | `#1` |
| 중심 위치 | `7시 방향 · r=0.82 · edge` |
| 라벨 수 | 클러스터에 속한 annotation 수 |
| **패널 수** | 서로 다른 `panel_id` 수 ← 가장 중요한 지표 |
| 패널 점유율 | `패널 수 / 전체 패널 수` |
| 주요 불량 | 클러스터 내 최빈 `defect_name` |
| 편중 Lot / 설비 | 해당 클러스터 패널의 Lot·설비 분포 중 전체 분포 대비 유의하게 과대표된 값 |

`라벨 수`가 아니라 **`패널 수`로 정렬한다.** 한 패널에 불량 20개가 몰린 것과, 20개 패널에 같은 자리 불량이 하나씩 있는 것은 완전히 다른 문제다. 후자가 공정 이슈다.

클러스터 행을 클릭하면 기여한 패널 썸네일이 그리드로 펼쳐진다.

### 4.3 불량명 × Region Matrix

`defect_name` × `{center, mid, edge}` 교차표.

각 셀에 관측 수와 함께 **면적 보정 잔차**를 표시한다. 균등 분포 가정이 아니라 **면적 비례를 귀무가설**로 둔다.

```text
기대 비율:  center = 0.35²           = 0.1225
            mid    = 0.75² - 0.35²   = 0.4400
            edge   = 1.00² - 0.75²   = 0.4375

expected_ij   = row_total_i × area_frac_j
std_residual  = (observed - expected) / sqrt(expected)
```

`|std_residual| > 2` 인 셀을 강조한다. 해석은 이렇게 읽힌다.
`암점 大 × edge: std_residual = +3.4` → 암점 大는 면적 대비 외곽에 유의하게 몰려 있다.

전체 표에 대해 카이제곱 독립성 검정을 수행하고 `χ², dof, p-value`를 하단에 명시한다.
**기대도수 5 미만 셀이 20%를 넘으면 검정 결과를 표시하지 않고 "표본 부족" 문구로 대체한다.**

### 4.4 FPCB 6시 방향 집중도

각도 데이터는 순환 변수이므로 일반 히스토그램의 평균·표준편차가 성립하지 않는다. **원형 통계**를 쓴다.

```text
θ_i = angle_deg (라디안)
C = mean(cos θ),  S = mean(sin θ)
평균 방향  μ = atan2(S, C)
집중도     R̄ = sqrt(C² + S²)        # 0 = 완전 균등, 1 = 한 점 집중
Rayleigh 검정: Z = n × R̄²,  p ≈ exp(-Z)
```

표시:

- 24 섹터 로즈 다이어그램(극좌표 막대)
- 평균 방향 μ 를 화살표로 오버레이하고 시계 방향 시각으로 병기 (`평균 방향: 6.8시`)
- `R̄` 와 Rayleigh `p-value`. `p < 0.05` 이면 `방향성 유의` 배지
- FPCB 근방(6시 ± 30°) 라벨 비율을 별도 수치로 표기

`R̄ < 0.1` 이면 "특정 방향 편중 없음"으로 명시한다. 약한 편중을 유의미한 패턴으로 오독하는 것을 막는다.

## 5. Pattern Analysis

`R/G/B/W` × `defect_name` 교차 히트맵.

- 셀 값 = 해당 패턴 이미지에서 검출된 해당 불량의 annotation 수

  매칭 엔진이 이들을 억제(`docs/matching_engine.md` 4절 3번 규칙)하므로, 사람이 확인해야 할 잠재 누락 후보다.

패턴별 주요 불량을 한 줄 요약으로 제시한다. `W 패턴: 명점 최다 검출 (n=42)`

## 6. Lot / Equipment / Process Analysis

### 6.1 Lot별 불량률

- 가로 막대. `불량 패널 수 / Lot 내 패널 수`
- **Lot 내 패널 수가 10개 미만이면 비율을 회색 처리하고 `n` 을 병기한다.** 3개 중 1개 불량인 33%를 500개 중 165개인 33%와 같은 높이로 그리면 안 된다.
- 전체 평균 불량률을 기준선으로 그린다

### 6.2 설비별 불량률

동일 규칙. 설비별로 **주요 불량 유형**을 스택 막대로 분해한다.
특정 설비에서 특정 불량이 편중되면 그것이 곧 조치 대상이다.

### 6.3 공정별 불량 유형

`process_name` × `defect_name` 스택 막대.

### 6.4 날짜별 Trend

- 일 단위 라인 차트. 축1 = 패널 수(막대), 축2 = 불량률(라인)
- 7일 이동평균 옵션
- 데이터 포인트가 7개 미만이면 이동평균을 숨긴다

## 7. Rendering & Performance

서버가 없으므로 모든 집계는 브라우저에서 수행한다.

- 차트 라이브러리: **ECharts**. 극좌표 히트맵과 로즈 다이어그램을 기본 지원하고, 데이터 포인트 수만 개에서도 canvas 렌더링이 안정적이다. 단일 HTML 번들에 인라인한다.
- 집계는 `selectors/` 모듈의 순수 함수로 구현한다. UI 컴포넌트 안에서 집계하지 않는다.
- 앱 시작 시 IndexedDB에서 `annotation`·`panel_result`를 메모리로 한 번 로드하고, 이후 필터링은 메모리 배열 위에서 수행한다. 패널 1만·라벨 5만 기준으로 수십 MB이므로 충분히 가능하다.
- DBSCAN·카이제곱·원형 통계는 필터 조합을 키로 메모이즈한다. 재계산 목표 `< 500ms`.
- 라벨 수가 2만을 넘으면 hotspot 클러스터링을 **Web Worker**로 옮겨 UI 블로킹을 막는다.
- 썸네일은 lazy load하고, `URL.createObjectURL`로 만든 URL은 뷰포트를 벗어나면 `revokeObjectURL`로 해제한다. 해제하지 않으면 스크롤 중 메모리가 계속 증가한다.

## 8. Empty & Degenerate States

각 섹션은 아래 상황에서 차트 대신 명시적 문구를 보여준다. 빈 차트를 그리지 않는다.

| 상황 | 처리 |
|---|---|
| 필터 결과 패널 0개 | `조건에 맞는 패널이 없습니다` |
| 불량 라벨 0개 | 2·4·5절 숨김. `불량이 검출되지 않았습니다` |
| annotation < 10 | 4.2 hotspot 숨김. `표본 부족 — 최소 10개 라벨 필요` |
| 카이제곱 기대도수 부족 | 4.3 검정 결과 숨김, 교차표만 표시 |
| `R̄ < 0.1` | 4.4에 `특정 방향 편중 없음` 명시 |
| 학습 패널 < `knn.min_train_panels` | 헤더에 `kNN 비활성 — Rule 단독 판정 중` 배너 |
